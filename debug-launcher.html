Remote BT Assist Debug Launcher

Debug Configuration

This launcher helps you start debugging sessions for your Remote BT Assist application.

Note: To debug JSON with Comments files, you need to install the "JSON with Comments" extension from the VSCode Marketplace.

Available Debug Configurations:

  1. Full Stack: Vite + Chrome (Recommended) - Runs both the Vite server and Chrome browser together
  2. Launch Chrome against localhost - Launches Chrome browser for debugging the frontend
  3. Debug Current File - Debugs the currently active file in VSCode
  4. Debug Vite Dev Server - Starts and debugs the Vite development server

Quick Actions

Click on the buttons below to start debugging:

Start Full Stack Debugging Start Vite Server Launch Chrome

Manual Steps

If the buttons don't work, follow these steps for full stack debugging:

  1. Open Debug panel in VSCode (Ctrl+Shift+D or Cmd+Shift+D)
  2. Select "Full Stack: Vite + Chrome" from the dropdown at the top
  3. Click the green play button or press F5

This will start both the Vite server and Chrome browser together, providing a complete debugging experience.

External Device Access

The Vite server is configured to be accessible from other devices on your network:

  1. Start the Vite server using one of the methods above
  2. Look for the Network URLs displayed in the terminal
  3. Use these URLs to access your app from other devices on the same network

Note: Make sure your firewall allows connections to port 8080 for external devices to connect.

Troubleshooting

If you encounter the "You don't have an extension for debugging 'JSON with Comments'" error:

  1. Click "Find 'JSON with Comments' extension" in the dialog
  2. Install the recommended extension
  3. Restart VSCode
  4. Try debugging again
